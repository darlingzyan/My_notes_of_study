# 算法竞赛：知识点与错题本联动体系构建指南



### 1. 记录格式：四步复盘法
每一道高质量的错题，都必须包含以下四个维度：

* **☠️ 死因 (Cause of Death)**：直白记录当时卡在哪里。是思路方向错了？还是二分边界写挂了？或是数据范围溢出了？
* **💡 破局点 (Key Insight)**：这道题最核心、最让你“拍大腿”的那一步逻辑转换。（通常只有一句话，比如“奇偶不变性”或“时光倒流处理”）。
* **⚠️ 避坑指南 (Trap Alerts)**：代码实现中极其容易忽略的细节（如 `long long` 溢出、离散化时忘记把 $0$ 加进去等）。
* **💻 核心代码 (Core Logic)**：**千万不要贴几百行的全量代码！** 只保留 `solve()` 函数和最关键的算法逻辑，去除所有冗余的头文件和快读模板。

### 2. 应用场景
* **赛前热身**：比赛前 30 分钟，不要再看新题，快速浏览最近 10 道错题的“破局点”和“死因”，唤醒大脑对常见陷阱的警觉。
* **同类归纳**：当你发现连续几道题都死于“没有使用倒序处理”时，这就是你必须去补充的战略武器。

## 二、 知识点总结：武器库的应用与记录策略

知识点总结记录的是**“通用武器的说明书”**。它的核心目的是**快速检索与模板复用**。

### 1. 记录格式：字典化结构

* **📖 通俗定义**：用一句话“说人话”解释这个算法（例如：二维偏序就是“HR 招聘看两科成绩”）。
* **🛠️ 标准模板**：经过实战反复验证的、绝对无错的、可以直接复制粘贴的 `struct` 或函数模板（例如树状数组、线段树、线性筛）。
* **🔍 适用标志**：看到题目出现什么“关键词”，就该掏出这个武器？（例如看到“最大值最小化”秒想二分答案）。

### 2. 应用场景
* **字典检索**：日常刷题或赛后补题时，遇到需要某种操作（如区间修改），一秒翻到对应模板，避免临场默写导致 Bug。
* **心法回顾**：定期复习 MEX 的定义、异或的周期性质等纯数学规律，确保基础理论无死角。

## 三、 双剑合璧：两者如何联动整理？

**联动是这套系统的灵魂！** 孤立的知识是死的，只有和错题相互印证才是活的。

### 1. 向上提炼：从“错题”中抽取“心法”
当你在一道错题中发现了一个非常通用的技巧时，绝不能只把它留在错题本里。
> **💡 实操案例**：
> 1. 在错题本写完《We are the Lights》这道题后，发现“倒序处理”极其好用。
> 2. 立即打开《知识点总结》，新增【离线倒序处理】这一独立章节。
> 3. 在知识点下注明：**实战案例见错题本 -> We are the Lights**。

### 2. 向下索引：给“武器”增加“实战案例”
当你在学习一个新的算法模板时，通过超链接或文字注明它解决过哪些硬仗。
> **💡 实操案例**：
> 1. 在《知识点总结》中录入了 `树状数组 (BIT)` 模板。
> 2. 在模板代码下方补充：**实战案例 -> HH的项链 (区间去重)、Robotic Rush (二维偏序)**。

### 3. 学习闭环
- **刷题** ➡️ 发现死角 ➡️ **记录错题本**
- **复盘错题** ➡️ 提炼通用技巧 ➡️ **更新知识点总结**
- **打比赛/遇新题** ⬅️ 检索模板与思路 ⬅️ **查阅知识点总结**

## 四、 Typora 高效笔记规范 (防崩溃指南)

为了保持笔记的绝对清爽和渲染的稳定，请严格遵守以下 Typora 使用规范：

1. **绝对放弃 HTML 折叠标签**：不要再使用 `<details>` 等标签，它们会导致 Typora 的代码高亮碎裂。用四级标题 `####` 分割代码即可。
2. **善用大纲视图 (Outline)**：保持 Typora 左侧大纲栏常开。严格按照 `## 一级分类` -> `### 二级题目` 的层级记录，实现一键跳转。
3. **隔离代码块**：在包裹代码的 ` ```cpp ` 和 ` ``` ` 前后，**必须保留空行**，确保渲染引擎不误判。
4. **内联数学公式**：利用 `$...$` 记录代数转换（例如 $S_R - S_{L-1} < t$ 移项变为 $S_{L-1} > S_R - t$），比纯文本更严谨、更具可读性。
5. **视觉切割**：在每一道错题、每一个大知识点结束时，敲三个减号 `---` 生成分隔线，让大脑在阅读时有喘息的空间。
